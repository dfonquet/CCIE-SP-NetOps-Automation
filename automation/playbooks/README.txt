Purpose:
  Stores Ansible playbooks that execute the automation lifecycle.

What belongs here:
  Render, validate, dry-run, deploy, diagnostics, and post-validation
  playbooks.

How it is used:
  Run these playbooks from the automation host after dependencies and
  inventory are ready.

What you can add next:
  Add service-specific playbooks, per-router batch controls, guarded rollback,
  and richer post-check assertions.
